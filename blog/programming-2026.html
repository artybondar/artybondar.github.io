Programming Today:
What's Actually Happening
in 2025–2026

AB
Arty Bondar
Fullstack Developer
May 2026Published
8 minRead time

Every few years the industry resets the rules. Right now we're in one of those moments — not slowly, but all at once. This is a grounded, data-driven look at where programming actually is in 2025–2026, stripped of the marketing noise.

I've been in this industry for over eight years, through PHP monoliths, the JavaScript explosion, the containerization wave, and now this. Each era felt big. This one feels different — because the tool that's changing is the act of writing code itself.

The Language Landscape

The rankings tell a clear story: Python won the AI era and dragged the whole industry with it. But the nuance is more interesting than a simple leaderboard.

Python's rise is structural, not cyclical. According to the Stack Overflow Developer Survey 2025, Python saw a seven percentage point jump in adoption in a single year — the largest single-year gain of any major language ever recorded in that survey. It also surpassed JavaScript as the most-used language on GitHub, driven almost entirely by AI and data science activity.

TypeScript quietly took over

While Python gets the headlines, TypeScript made the quieter, more permanent conquest of everyday web development. 69% of developers now use TypeScript for large-scale web applications, and it replaced Java in GitHub's top three languages. The JavaScript ecosystem didn't die — it grew up.

Rust: most loved, least hired

For the tenth consecutive year, Rust topped the Stack Overflow "most admired" list at 72% approval. Amazon, Microsoft, and Cloudflare have integrated it into critical infrastructure. Yet corporate adoption still lags community enthusiasm — the steep learning curve keeps it a specialist tool. Rust job postings grew 35% year-over-year in 2025, which sounds impressive until you realize the base was small. This is a language for the long game.

PHP isn't dead (stop saying that)

Over 75% of websites that declare a server-side language still run PHP. Laravel modernized the ecosystem considerably, and an enormous volume of production systems need daily care. It may not be the default choice for greenfield work in 2026, but dismissing it is a luxury most of us can't afford.

AI Tools: The Real Numbers

This is where things get genuinely complicated. The productivity claims are everywhere. The reality is more nuanced.

The top-line numbers look transformative. But a randomized controlled trial by METR, running 16 experienced developers through 246 tasks in 2025, found something surprising: experienced developers actually took 19% longer when using AI tools, largely due to review and correction time. The gains concentrated in junior developers and repetitive tasks.

// Key insight

AI tools are a force multiplier for experience, not a substitute for it. In well-structured teams they reduce incidents by 50%. In poorly structured ones, they create twice as many. The difference is engineering culture, not the tool.

A McKinsey study from February 2026 surveying 4,500 developers found that AI tools cut time on routine coding tasks by an average of 46%. That number is real — for boilerplate, test generation, and restructuring. For complex architectural decisions, novel problem-solving, and system design, the current tools still follow rather than lead.

"For simpler tasks like restructuring the code base and writing tests, AI-powered workflows have achieved speedups of up to 90%. But gains are more modest for other tasks, and the disruption caused by overhauling existing processes often counteracts the increased coding speed."

— Rob Witoff, Head of Platform, Coinbase · MIT Technology Review, 2026

What this means for developers

Job postings requiring experience with AI coding tools increased 340% between January 2025 and January 2026. Meanwhile, pure implementation roles — writing spec-to-code — declined 17%. The shift isn't elimination, it's elevation. The developers thriving are the ones who review and direct AI output, not the ones competing with it.

The Modern Stack

Beyond languages, the infrastructure story of 2025–2026 is consolidation. The chaotic microservices boom is settling into pragmatic patterns.

  • PostgreSQL has been the most desired and admired database for three consecutive years in Stack Overflow surveys. Redis usage jumped 8% in a single year as applications demand in-memory caching at every layer.
  • Docker + Kubernetes remain the baseline. The tooling debate has largely ended — the question now is who manages it (platform team vs cloud vs third-party).
  • FastAPI (Python) saw a 5-point usage jump in 2025, the largest single shift in the web framework space. Python for APIs is no longer a surprising choice.
  • Edge computing is maturing from concept to practice. Cloudflare Workers, Vercel Edge Functions, and Deno Deploy are production-grade. Reducing latency at the network edge is now a viable first-class strategy.
  • WebAssembly (WASM) is quietly becoming serious infrastructure. Compiled Rust and Go modules running in the browser with near-native performance is no longer experimental.

The Job Market

The feared mass displacement from AI has not materialized — yet. Global developer employment reached 28.7 million in 2025, a new high, with the US adding approximately 72,000 jobs. But the composition is shifting.

What pays: Python leads US hiring demand with over 64,000 open positions. Go and Solidity command salaries above $150,000 in US markets. C# was TIOBE's language of the year in 2025 — Microsoft's .NET ecosystem matured significantly for cross-platform work, and enterprise demand reflects that.

What's contracting: Junior implementation roles are under real pressure. A Stanford study found employment among software developers aged 22–25 fell nearly 20% between 2022 and 2025. Entry-level positions that were previously training grounds are being compressed by AI that can produce working first drafts instantly.

// For junior developers

The path forward is the same as always, accelerated: build real things, ship them, understand them deeply. The ability to review, debug, and improve AI-generated code is now a core skill, not a bonus one. Learn to evaluate output, not just produce it.

My Take

After eight years in this industry, the most dangerous thing I see isn't AI replacing developers. It's developers misreading which skills to compound.

The fundamentals — algorithms, data structures, system design, debugging, clear thinking — have never mattered more, because they're exactly what AI can't reliably do yet. AI is a brilliant autocomplete. The programmer's job is increasingly to ask the right questions, validate the answers, and understand deeply enough to know when the confident output is wrong.

The language and tool bets I'd make today: Python if you're near the AI/data space, TypeScript if you're building products for the web, Rust if you're doing systems work and have the patience for it. And SQL — always, everywhere, non-negotiable.

Summary

Python dominates AI/data and is now the top language by GitHub activity. TypeScript is the default for serious web development. Rust is the most admired language but still a specialist tool.

AI tools write 41% of code and save ~3.6h/week on average, but experienced developers need to review carefully — gains are real but uneven. The job market is growing, but the nature of roles is shifting upward.

The developers who thrive in 2026 are the ones who understand the machine well enough to direct it — not just the ones who can prompt it.
